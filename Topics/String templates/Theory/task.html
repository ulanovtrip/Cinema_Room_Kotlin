Sometimes you need to put the values of variables in a text. Lucky you, Kotlin allows you to do that. Meet string templates. In this lesson, we will find out what it is and why they are the best.

Templates for variables

Suppose we need to display a message about today's temperature (in Celsius) in some city.

Now, the temperature in ... is ... degrees Celsius.

Instead of ... we need to write certain values.

If we have two variables city and temp, we can build the result string using a sequence of concatenations:

val city = "Paris"
val temp = "24"

println("Now, the temperature in " + city + " is " + temp + " degrees Celsius.")

Of course, this code is simple enough and works correctly. Is it enough? Nope, for it is not very concise.

Kotlin provides a more convenient way to do the same thing. The way is string templates. How to use it? To put a value of a variable to a string, write the dollar sign $ before the variable's name.

val city = "Paris"
val temp = "24"

println("Now, the temperature in $city is $temp degrees Celsius.")

This code is more readable than the one with concatenations.

"Trust no one, homie", a rapper once said. Check for yourself that both code snippets print the same message:

Now, the temperature in Paris is 24 degrees Celsius.

If we do not want to print a string, we can create a variable and use it:

val value = "55"
val currency = "dollars"
val price = "$value $currency" // "55 dollars"

String templates are a recommended way to build strings based on the values of variables. Try to use it instead of string concatenation.

Templates for expressions

You can use string templates to put a result of an arbitrary expression in a string. To do that, include the entire expression in curly braces {...} after the dollar sign $. For example:

val language = "Kotlin"
println("$language has ${language.length} letters in the name")

It prints:

Kotlin has 6 letters in the name

Here {language.length} is an expression to evaluate. If you skip the braces, it will output something different:

Kotlin has Kotlin.length letters in the name

So, always use curly braces for expressions in string templates to avoid such mistakes. But do not add them if you want to output only a value of a variable even though it works.

Idiom

In this course, you will learn some of Kotlin's idioms. Idioms are templates for clear and readable code. It's important for other people to be able to understand your code, so you should utilize these patterns. All idioms are endorsed by the community, so using them will bring you closer to Kotlin-style code. You can find a list of the most common idioms at kotlinlang.org.

The first idiom we'll look at is the use of string templates to compose strings.

val language = "Kotlin"
println("Have a nice $language!")        // nice code
println("Have a nice " + language + "!") // bad practice

Conclusion

What to remember from this lesson? There are two main points:

  1. Use string templates to insert values of variables into a string like "this string has $value".
  2. If you would like to get an arbitrary expression, use curly braces like "The length is ${str.length}".
